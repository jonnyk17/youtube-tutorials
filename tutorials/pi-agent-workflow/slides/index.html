Section 2

What is harness engineering?

A short primer before the build

Mental model

Every agent has two parts

Model Decides

The reasoning engine. Probabilistic. Decides what should happen next, given the context.

Harness Does

The software around the model that lets it act on the world. Gathers context. Executes tool calls. Manages state. Decides what's possible.

agent = model + harness
Mechanism

The agent loop

Every harness implements this loop. The interesting differences live in how each step is governed — what context goes in, what tools are exposed, what's allowed.

Responsibilities

What the harness handles

Context
  • Gathers what the model sees
  • Manages the context window
  • Compacts and summarizes
Action
  • Exposes tools to the model
  • Executes tool calls
  • Manages permissions
Lifecycle
  • Runs hooks at key events
  • Enforces guardrails
  • Decides when to stop
Why harnesses exist

Every harness piece exists for a reason

Tools
←
Model can't act on the world
Permissions
←
Model can ask for destructive things
Retries
←
Model produces broken work sometimes
Evaluators
←
Models skew positive on their own work
Context mgmt
←
Conversations overflow the window

As models improve, some pieces stop being load-bearing — others have to be added to reach the new ceiling. Harnesses don't shrink as models get better. They move.
