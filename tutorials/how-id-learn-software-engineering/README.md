# How I'd Learn Software Engineering (When AI Writes the Code)

Companion repo for the video. The complete roadmap as a reference you can actually use.

## The Map

Eight areas of competence. The first six are timeless. The last two are about how the game has changed.

```
How I'd Learn Software Engineering
│
├── 1. Core Fundamentals
│   How code works. One language deeply. Reading code fluently.
│
├── 2. System Design
│   How to put systems together. The metaskill AI can't replace.
│
├── 3. Product Thinking
│   Requirements, users, deciding what to build before anything gets coded.
│
├── 4. The Development Process
│   Spec, plan, build, review, refactor, debug, commit. How software gets made.
│
├── 5. Operations
│   Deployment, Linux, monitoring. How software runs in the real world.
│
├── 6. Communication
│   Writing clearly, explaining decisions, the human layer around the code.
│
├── 7. AI Tools & Workflows
│   Context engineering, spec-driven prompting, code review workflows.
│
└── 8. Deliberate Practice
    Build real things. Ship end to end. Sometimes work without AI.
```

## What I'd Skip

- **Leetcode grinding.** AI solves these faster than you ever will.
- **Memorizing framework APIs.** Understand the concepts. Let AI handle syntax.
- **Deep CS theory.** Learn the practical subset and move on.
- **Tutorial hell.** Start building real things as soon as you understand the basics.

These aren't useless. The return on investment has changed. Spend your hours where they compound most.
