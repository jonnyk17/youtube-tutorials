1. AI Review vs Human Review

Talk through:

AI and human review are complementary, not competing. AI handles the volume. Humans handle the judgement. Five minutes of human review focused on the right things beats thirty minutes of line-by-line reading.

AI Review

Superior at: pattern matching, speed, consistency
  • Security scans (OWASP, injection, secrets, XSS)
  • Bug detection (logic, null, races, off-by-one)
  • Style guide adherence
  • Syntax and completeness
  • Scales to any volume and large PRs
  • Never gets tired, never skips
Handles the mechanical checks at machine speed

Human Review

Superior at: judgement, context, strategy
  • Does this solve the business problem?
  • Is this the right architecture?
  • Does it meet acceptance criteria?
  • Business logic verification
  • Will this cause problems in 3 months?
  • Should we even build this?
5 min focused on the right things > 30 min reading every line

3. What Good Review Looks For

Talk through:

Before the tools, ground the viewer in what a human reviewer actually checks. These four categories apply at every level. AI makes different mistakes than humans: it over-engineers, adds unnecessary abstractions, and silently changes behavior while "fixing" something.

Correctness

  • Does this code do what it's supposed to do?
  • Are edge cases handled? (empty, null, zero, negative)
  • Are error paths correct, not just present?

Intent & Blast Radius

  • Does this change do only what was asked?
  • Did the agent touch files it shouldn't have?
  • #1 issue with AI code: silent side effects

Security

  • Input validation at system boundaries
  • No credentials or secrets in code
  • No injection vectors (SQL, command, XSS)

Design

  • Is this the simplest solution that works?
  • Is there unnecessary abstraction?
  • Would a new team member understand this?

4. What Each Level Catches

Talk through:

Each level catches different things at different cost. Levels 1-3 are where 90% of the value is, and they're all free. Start at Level 1, add layers as your codebase grows.

Level What it catches Cost / Effort
Gates Lint errors, type errors, formatting Free. Automatic.
1 Unexpected changes, wrong files, scope creep Free. 30 seconds.
2 Obvious bugs, over-engineering, unused code Free. 10 seconds.
3 Project-specific rule violations Free. Your rules.
4 Security vulnerabilities, SAST, coverage gaps Free/Paid. Local.
5 Architectural bugs, cross-file issues Free/Paid. On PR.
Human Business logic, architecture, "should we build this?" Time. Judgement.
